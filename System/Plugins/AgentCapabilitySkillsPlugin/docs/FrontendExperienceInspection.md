# FrontendExperienceInspectionSkill

## 简述

用于前端 UI、交互、布局、状态、动画、上传反馈和视觉体验问题。重点是找到状态来源、渲染边界和样式约束。

## 何时使用

当用户描述截图、界面不好看、展开抖动、按钮状态异常、进度缺失、文件图标不准确、布局留白或交互卡顿时使用。

## 不要使用的情况

纯后端协议、数据库、模型调用或无 UI 表现的问题优先使用其它技能。

## 执行指引

1. 先定位组件、状态来源、事件处理和样式文件，不要只改视觉表层。
2. 区分文档流问题、浮层问题、状态刷新问题、滚动容器问题和数据计算问题。
3. 有截图时先提取可观察症状；没有截图时从用户描述生成检索词定位组件。
4. 修改布局前确认父容器、滚动区域、定位方式、响应式约束和可访问交互。
5. 对动态文本、上传进度、折叠面板、文件列表这类 UI，确认 loading、success、error、empty、overflow 状态。
6. 完成后优先做构建或针对组件的测试；有视觉风险时说明需要截图验证。

## 证据标准

结论需要读到组件实现或样式约束。涉及状态异常时，需要读到事件流或数据源。

## 完成标准

回答应说明 UI 问题根因、涉及组件、交互状态、改动点和验证方式。实现时避免让展开内容影响主文档流，除非设计明确要求。
