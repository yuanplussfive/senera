# MemoryFormationSkill

## 简述

用于长期记忆、用户画像、偏好、项目知识和关系网络设计。重点是候选记忆、证据链接、合并策略和召回投影。

## 何时使用

当用户讨论长期记忆、用户偏好、画像自动完善、知识点学习、关系形成、记忆压缩或跨会话召回时使用。

## 不要使用的情况

短期任务执行、一次性代码修改或没有持久化价值的闲聊，不应生成长期记忆。

## 执行指引

1. 先区分信息类型：用户身份、稳定偏好、项目规则、任务事实、工具证据、新知识、关系边。
2. 只把稳定、重复、可复用或用户明确确认的信息作为候选记忆。
3. 无意义会话、临时情绪、一次性中间状态、未验证猜测和敏感密钥不进入长期记忆。
4. 每条候选记忆必须链接来源：用户原话、工具证据、artifact 或多次行为统计。
5. 生成关系时使用实体、概念、属性和证据边；例如“香蕉”和“苹果”可通过共同上位概念“水果”形成关系，但应标注关系来源。
6. 召回时按任务需要投影最小证据，不把完整历史塞回上下文。
7. 冲突时保留时间、来源和置信信息，优先使用用户最新明确更正。

## 证据标准

用户偏好需要来自明确表达或重复行为；项目知识需要来自仓库、工具结果或用户确认；概念关系需要可解释来源。

## 完成标准

输出应给出候选记忆结构、过滤规则、关系形成策略、召回投影和避免污染长期记忆的约束。
