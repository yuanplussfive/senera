# Execution Resource Inspect

## 简述

读取后台执行资源的当前状态和从指定游标之后保留的事件，不进行轮询等待。

## 何时使用

需要立即确认进程状态、退出码、累计输出或恢复断线后的事件时使用。

## 不要使用的情况

需要等待新输出时使用 ExecutionResourceWait；不要重复高频调用本工具轮询。

## 输入

`resourceId` 来自 ShellStartTool。`cursor` 是已经消费的最后事件游标，省略时返回当前仍保留的全部事件。

## 输出

返回状态、最新游标、最早保留游标、截断标记、增量事件、退出码和信号。

## 执行约束

只能读取当前会话或当前请求拥有的资源。缓冲超过预算时最旧输出会被淘汰，并通过 `truncated` 明确标记。
